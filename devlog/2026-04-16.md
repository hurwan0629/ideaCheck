# 2026-04-16 | DB 모델 정리 및 processor 구현

## 결정한 것

- `MarketRawSource`에서 `content_type` 제거
  - raw 저장 시점엔 AI 분석 전이라 내용 분류 불가
  - AI 분석 결과는 `MarketExtract.extract_type`에 이미 있어 중복
- `MarketRawSource.source_name` → `title`로 교체
  - source_name은 뉴스 매체명을 의도했으나 크롤러가 제공하지 않음
  - 실제로 필요한 건 기사 제목이라 title로 대체
- `MarketRawSource`에 `published_at` 추가
  - 크롤러가 기사 발행일을 주는데 저장할 컬럼이 없었음
- `MarketExtract.extracted_data` → `JSONB`로 통합
  - `summary(JSONB)` + `extracted_data(Text)` 두 컬럼이 있었는데, AI가 어차피 JSON으로 반환하므로 `extracted_data: JSONB` 하나로 통합
- SQLAlchemy 모델에 `MappedAsDataclass` 적용 (market_raw_sources, market_extracts, competitor_policies)
  - 런타임에 동적으로 생성되는 `__init__`을 IDE가 인식 못 하는 문제 해결
  - `init=False`: PK, timestamp 등 DB 자동 생성 필드는 생성자에서 제외
  - dataclass 규칙상 default 없는 필드를 앞에 배치

## 고려했던 것들

- `content_type`을 raw에 남기고 AI 처리 후 UPDATE하는 방식
  - → raw는 수집 원본 그대로 보존하는 테이블이라 나중에 UPDATE하는 게 어색함. extract 테이블에 분류 결과를 두는 게 책임 분리상 맞음
