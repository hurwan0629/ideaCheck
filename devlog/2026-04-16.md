# 2026-04-16 | DB 모델 정리 및 processor 구현

## 결정한 것

- `MarketRawSource`에서 `content_type` 제거
  - raw 저장 시점엔 AI 분석 전이라 내용 분류 불가
  - AI 분석 결과는 `MarketExtract.extract_type`에 이미 있어 중복
- `MarketRawSource.source_name` → `title`로 교체
  - source_name은 뉴스 매체명을 의도했으나 크롤러가 제공하지 않음
  - 실제로 필요한 건 기사 제목이라 title로 대체
- `MarketRawSource`에 `published_at` 추가
  - 크롤러가 기사 발행일을 주는데 저장할 컬럼이 없었음
- `MarketExtract.extracted_data` → `JSONB`로 통합
  - `summary(JSONB)` + `extracted_data(Text)` 두 컬럼이 있었는데, AI가 어차피 JSON으로 반환하므로 `extracted_data: JSONB` 하나로 통합
- SQLAlchemy 모델에 `MappedAsDataclass` 적용 (market_raw_sources, market_extracts, competitor_policies)
  - 런타임에 동적으로 생성되는 `__init__`을 IDE가 인식 못 하는 문제 해결
  - `init=False`: PK, timestamp 등 DB 자동 생성 필드는 생성자에서 제외
  - dataclass 규칙상 default 없는 필드를 앞에 배치

## 고려했던 것들

- `content_type`을 raw에 남기고 AI 처리 후 UPDATE하는 방식
  - → raw는 수집 원본 그대로 보존하는 테이블이라 나중에 UPDATE하는 게 어색함. extract 테이블에 분류 결과를 두는 게 책임 분리상 맞음

# 2026-04-16 (3) | competitor_crawler 완성

## 결정한 것

- JS 필요 여부를 `needs_js` 컬럼 대신 자동 감지로 처리
  - httpx로 먼저 시도 → SPA 패턴/noscript/body 텍스트 길이로 판단 → Playwright 폴백
  - 경쟁사 수가 적어 DB 캐싱 없이 매 분기 재탐지 해도 부담 없음
- 경쟁사 데이터 추출을 AI 한 번 호출로 통합 (description, target_customer, type, features)
  - 사이트마다 selector가 달라 하드코딩 불가 → AI가 body_text 보고 구조화
  - HTML 통째 넘기지 않고 `get_text()`로 텍스트만 추출 → 토큰 절약
- `type` 카테고리를 고정 목록으로 프롬프트에 명시
  - 자유 텍스트로 두면 "핀테크"/"FinTech"/"금융서비스" 등 불일치 발생
- `browser_client`를 `clients.py` 싱글턴으로 관리
  - 매 요청마다 httpx.Client 새로 생성하지 않고 커넥션 풀 재사용
  - User-Agent 등 브라우저 헤더 설정으로 1단계 봇 차단 우회

## 고려했던 것들

- URL만 보고 JS 필요 여부 판단 → AI도 URL만으론 불가, 의미 없음
- AI로 JS 필요 여부 판단 → HTML 받아서 휴리스틱으로도 충분, API 비용 낭비
- `needs_js` 컬럼 수동 관리 → 확장성 없어 자동 감지 채택
- CompetitorFeature upsert → AI가 기능명을 매번 조금씩 다르게 쓸 수 있어 기준 불명확, delete→insert가 단순함

## 완료한 작업

- `clients.py`에 `browser_client` 추가 (브라우저 헤더 위장 httpx.Client)
- `competitor_crawler.py` 전면 재작성
  - `_fetch_with_httpx` / `_fetch_with_playwright` → BeautifulSoup 반환으로 fetch/extract 분리
  - `_needs_js` 자동 감지 (SPA 패턴, noscript, body 텍스트 길이)
  - `_extract_with_ai` AI 기반 구조화 추출
  - `_update_competitor` 저장 코드 (Competitor 갱신 + CompetitorFeature delete→insert)
